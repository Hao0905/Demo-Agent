## 4. Tạo và cấu hình Agent

### 4.1 Cấu trúc file YAML

```yaml

name: my_agent # Bắt buộc - tên agent

model: gemini-2.5-flash # Model LLM

instruction: | # System prompt
  Bạn là trợ lý ảo chuyên nghiệp.
  Hãy trả lời một cách thân thiện và chính xác.

tools: # Tools (tuỳ chọn)
  - name: google_search
    type: google_search
    description: Tìm kiếm thông tin trên web

sub_agents: # Sub-agents (tuỳ chọn)
  - name: writer
    model: gemini-2.5-flash
    instruction: Viết nội dung dựa trên yêu cầu.
    mode: single_turn
    output_schema: # Schema đầu ra có cấu trúc
      type: object
      properties:
        title:
          type: string
        content:
          type: string
```

### 4.2 Các bước tạo Agent qua Dashboard (@[HDSD Agent Platform](mention://4911eafa-7777-4790-bebe-67b7e51c45bf/document/a7a184d8-3620-43db-9fb4-3d25be4b7810))

**Bước 1:** Vào Dashboard → **Agents** → **New Agent**

**Bước 2:** Soạn YAML config (xem [Hướng dẫn cấu hình chi tiết](https://./AGENT_RUNTIME_GUIDE.md))

**Bước 3:** Click **Validate** để kiểm tra cấu hình

**Bước 4:** Click **Create & Deploy**

**Bước 5:** Đợi status chuyển từ `creating` → `running` (kiểm tra ở Fleet Dashboard)

### 4.3 Tạo Agent qua API

```bash

curl -X POST http://localhost:8000/agents/v2/from-yaml \
  -H "X-API-Key: $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "config_yaml": "name: my_agent\nmodel: gemini-2.5-flash\ninstruction: Bạn là trợ lý ảo.",
    "replicas": 1
  }'
```

Response:

```json
{
  "id": "uuid-deployment-id",
  "name": "my_agent",
  "status": "creating",
  "replicas": 1
}
```

### 4.4 Tham khảo cấu hình nhanh

| Field | Bắt buộc | Mô tả |
|----|----|----|
| `name` | ✅ | Tên agent |
| `model` | ❌ | Model LLM (mặc định: `gemini-2.0-flash-exp`) |
| `instruction` | ❌ | System prompt |
| `llm` | ❌ | Cấu hình LLM đầy đủ (base_url, api_key, temperature...) |
| `agent_type` | ❌ | `llm` (mặc định), `workflow`, `dynamic_workflow` |
| `tools` | ❌ | Danh sách tools |
| `sub_agents` | ❌ | Sub-agents và workflow nodes |
| `workflow` | ❌ | Định nghĩa edges (khi `agent_type: workflow`) |
| `output_key` | ❌ | Key trong session state để lưu kết quả |

### 4.5 Các loại Agent

#### LLM Agent (mặc định)

Agent đơn giản, có thể delegate tasks cho sub-agents qua natural language.

```yaml

name: assistant

model: gemini-2.5-flash

instruction: Bạn là trợ lý đa năng.
tools:
  - name: calculator
    type: function
    code: |
      def calculator(a: float, b: float, op: str) -> float:
          if op == "+": return a + b
          if op == "-": return a - b
          return a * b
```

#### Workflow Agent

Đồ thị thực thi xác định với edges rõ ràng.

```yaml

name: content_pipeline

agent_type: workflow

workflow:
  edges:
    - ["START", planner, writer, reviewer]
sub_agents:
  - name: planner
    model: gemini-2.5-flash
    instruction: Lên kế hoạch nội dung.
    mode: single_turn
  - name: writer
    model: gemini-2.5-flash
    instruction: Viết nội dung dựa trên kế hoạch.
    mode: single_turn
  - name: reviewer
    model: gemini-2.5-flash
    instruction: Review và cải thiện nội dung.
    mode: single_turn
```

#### Dynamic Workflow Agent

Code-driven orchestration — cho phép loop, rẽ nhánh, parallel execution.

```yaml

name: research_agent

agent_type: dynamic_workflow

dynamic_workflow:
  entrypoint: pipeline
  nodes:
    - name: pipeline
      code: |
        import asyncio
        async def pipeline(ctx, topic: str):
            results = await asyncio.gather(
                ctx.run_node(researcher_a, topic),
                ctx.run_node(researcher_b, topic),
            )
            return await ctx.run_node(synthesizer, results)
sub_agents:
  - name: researcher_a
    model: gemini-2.5-flash
    instruction: Nghiên cứu từ góc nhìn A.
    mode: single_turn
  - name: researcher_b
    model: gemini-2.5-flash
    instruction: Nghiên cứu từ góc nhìn B.
    mode: single_turn
  - name: synthesizer
    model: gemini-2.5-flash
    instruction: Tổng hợp kết quả nghiên cứu.
    mode: single_turn
```

### 4.6 Mode cho Sub-Agent

| Mode | `sub_agents`? | Hành vi |
|----|----|----|
| `chat` | ✅ Có | Full conversation, delegate tự do |
| `task` | ❌ Không | Có thể hỏi clarifying questions, auto-return |
| `single_turn` | ❌ Không | Một lần thực thi, không hỏi lại |

### 4.7 Tool types

| Type | Mô tả |
|----|----|
| `function` | Code Python inline |
| `mcp_stdio` | MCP server qua stdio |
| `mcp_http` | MCP server qua HTTP |
| `mcp_sse` | MCP server qua SSE |
| `openapi` | OpenAPI toolset |
