## 1. Tổng quan

### Agent Provider là gì?

Agent Provider là nền tảng multi-tenant cho phép bạn triển khai và quản lý AI Agent trên Kubernetes. Với tư cách là nhà phát triển ứng dụng, bạn có thể:

* **Tạo Agent** từ file cấu hình YAML — xác định model, instruction, tools, sub-agents
* **Triển khai Agent** lên Kubernetes với một API call — platform lo phần hạ tầng
* **Tương tác với Agent** qua real-time streaming (SSE) hoặc request-response
* **Đánh giá chất lượng Agent** với bộ test dataset
* **Tinh chỉnh instruction** qua Prompt Lab mà không cần redeploy

### Kiến trúc tổng quan

```
┌──────────────────┐        ┌──────────────────┐       ┌──────────────┐
│  Ứng dụng bạn     │──────>│  Agent Provider  │──────>│  Kubernetes  │
│  (Frontend App)   │ HTTP  │  Orchestrator    │  K8s  │  Cluster     │
│                   │<──────│  (FastAPI)       │<──────│              │
└──────────────────┘  SSE   └──────────────────┘       └──────┬───────┘
                                                              │
                                                              v
                                                     ┌────────────────┐
                                                     │  Agent Runtime │
                                                     │  (ADK + A2A)   │
                                                     │  ┌──────────┐  │
                                                     │  │ LLM Agent│  │
                                                     │  │Workflow  │  │
                                                     │  │Sub-agent │  │
                                                     │  └──────────┘  │
                                                     └────────────────┘
```

### Luồng dữ liệu cơ bản


1. **Ứng dụng Frontend** gửi request HTTP tới Orchestrator API
2. **Orchestrator** xác thực API key, xác định tenant, chuyển tiếp request tới Agent Runtime
3. **Agent Runtime** xử lý request bằng ADK agent (có thể gọi LLM, tools, sub-agents)
4. **Kết quả** trả về qua SSE stream hoặc JSON response

## 2. Bắt đầu nhanh

### Yêu cầu

* API key tenant (lấy từ Dashboard > Settings hoặc từ admin)
* Deployment ID của Agent đã triển khai

### Lấy API key và Deployment ID

**Cách 1: QUA DASHBOARD**


 ![](/api/attachments.redirect?id=d3c3e732-836b-41ad-99ac-295f799d935d)



1. Vào Dashboard → **Settings** (icon bánh răng)
2. API key hiển thị ở mục "API Key". Nếu chưa có, tạo tenant mới qua admin.
3. Deployment ID lấy từ URL khi xem Agent: `/agents/[deployment_id]/status`
4. Hoặc vào **Fleet Dashboard** → click Agent → copy ID từ URL

**Cách 2: QUA API**

```bash
# Lấy danh sách agents -> có deployment_id

curl -H "X-API-Key: $API_KEY" http://localhost:8000/agents/

# Lấy thông tin tenant hiện tại

curl -H "X-API-Key: $API_KEY" http://localhost:8000/tenants/me
```

### Test nhanh

```bash
# Gửi task đơn giản (non-streaming)
curl -X POST http://localhost:8000/task/send \
  -H "X-API-Key: $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "deployment_id": "your-deployment-id",
    "user_id": "developer@myapp.com",
    "task_message": "Xin chào! Hãy giới thiệu về bạn."
  }'

# Gửi task với streaming

curl -N -X POST http://localhost:8000/task/stream \
  -H "X-API-Key: $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "deployment_id": "your-deployment-id",
    "user_id": "developer@myapp.com",
    "task_message": "Viết một đoạn giới thiệu ngắn về công ty ABC."
  }'
```

Kết quả trả về dạng JSON hoặc SSE stream tùy endpoint.

## 3. Dashboard

Dashboard là công cụ chính để quản lý Agent. Giao diện Next.js 16, giao tiếp với Orchestrator qua API proxy.


 ![](/api/attachments.redirect?id=702c0efd-c650-4eff-8a42-bec2b9fc7e14)


### 3.1 Fleet Dashboard (`/`)

Màn hình tổng quan toàn bộ fleet Agent:

* **Metric cards**: tổng số agent, session đang hoạt động, deploy issues
* **Attention needed**: agent không running cần xử lý
* **Fleet pulse**: agent đang chạy với link truy cập nhanh
* **Cluster Events**: sự kiện gần đây

### 3.2 Agent Registry (`/agents`)

Danh sách tất cả deployments:

| Feature | Mô tả |
|----|----|
| Danh sách Agent | Bảng đầy đủ: status, model, replicas |
| Inline editing | Sửa tên, mô tả trực tiếp |
| Lifecycle actions | Deploy, undeploy, redeploy, delete |
| Scale | Tăng/giảm số replicas |

### 3.3 Agent Configuration (`/agents/[id]/config`)


 ![](/api/attachments.redirect?id=1eceaf93-693a-4085-9ed9-d3d20aa528e3)


Trình soạn thảo YAML với Monaco Editor:

* Soạn thảo cấu hình Agent (YAML)
* Validate cấu hình trước khi deploy
* Reload config nóng (không downtime)
* Tạo Agent mới từ template

### 3.4 Chat (`/agents/[id]/chat`)


 ![](/api/attachments.redirect?id=fbce141b-ae00-4757-bc1a-c85855f27ba4)


Giao diện chat real-time với SSE streaming:

* Danh sách conversation
* Gửi message + nhận response stream
* Xem ADK trace (thinking, tool calls, delegation)
* File upload (multimodal)

### 3.5 Task Console (`/agents/[id]/tasks`) (deprecated)

A2A task composer:

* Soạn task message
* Stream execution real-time
* Event log (token, thinking, artifact, status)
* Trace inspector

### 3.6 Evaluation (`/agents/[id]/evaluate`)


 ![](/api/attachments.redirect?id=f23a7514-19dc-444e-a775-ef8f1c382fca)


Đánh giá chất lượng Agent:

* Upload dataset JSON
* Chạy evaluation job
* Xem kết quả pass/fail
* Raw JSON toggle

### 3.7 Prompt Lab (`/agents/[id]/prompt-lab`)


 ![](/api/attachments.redirect?id=863ef0c3-26a8-4cee-b190-fb5d372748a5)


Thử nghiệm instruction:

* Sửa instruction cho main agent và sub-agents
* Slider temperature/top-P
* Test với streaming response
* Sub-agent tree selector

### 3.8 Settings (`/settings/api`)

Cấu hình API key và User ID:

* Lưu trong localStorage (trình duyệt)
* Mặc định user ID: `operator@local`
